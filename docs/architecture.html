⚖️Governance Layer

Every project starts with explicit governance. Constitution defines principles, Framework defines standards.

📜 CONSTITUTION.md

Defines the immutable governance principles for the project. Principles are numbered (P-I, P-II, ...) and traced through the knowledge graph to requirements, tests, and tasks.

governance/CONSTITUTION.md

📐 FRAMEWORK-PRINCIPLES.md

Coding standards, architectural patterns, naming conventions, and quality thresholds. Referenced by all three plugins during their respective pipelines.

governance/FRAMEWORK-PRINCIPLES.md

📂 .specify/ Directory

Shared specification state: context.json, active feature tracking, knowledge graph, dashboard data, and RAG memory index. The single source of truth for pipeline state.

governance/.specify/

🏷️Evidence Protocol

Every claim in every artifact must carry an evidence tag. If more than 30% of claims are assumptions, a warning banner is mandatory.

📦 [CODE] ⚙️ [CONFIG] 📄 [DOC] 🔍 [INFERENCE] ⚠️ [ASSUMPTION]

Zero-Hallucination Rule: If a claim has severity=CRITICAL combined with [ASSUMPTION], it is flagged for mandatory human validation. Inferences are never presented as facts.

The core philosophy of the kit: intent is preserved from natural language all the way to running code, with cryptographic verification at each step.

Traceability: The knowledge graph connects every node in the chain. Requirements trace to tests. Tests trace to tasks. Tasks trace to code. Orphan detection runs automatically via the /sdd:graph command and the sentinel heartbeat.
