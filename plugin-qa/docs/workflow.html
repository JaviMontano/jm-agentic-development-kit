Workflow Guide

9 movements, 4 quality gates, MOAT methodology

9 Movements

Each movement maps to specific skills and quality expectations

1

IDEATE

Generate plugin concepts from natural-language descriptions. Define the plugin's purpose, target users, and high-level agent/skill topology.

/pqa:create · /pqa:ideate-component
2

PLAN

Architect the plugin structure: directory layout, agent roles, command tree, and MOAT depth strategy for each skill.

/pqa:plan-moat
3

DESIGN

Author detailed agent definitions and skill designs. Each agent gets a role, tools, and personality. Each skill gets MOAT structure.

/pqa:design-agent · /pqa:design-skill
4

SPECIFY

Write formal specifications: command triggers and parameters, skill MOAT requirements, hook event bindings, and manifest schemas.

/pqa:spec-agent
5

BUILD

Scaffold the complete plugin directory tree. Generate all files, MOAT assets, hook configurations, and the CLAUDE.md entry point.

/pqa:build
6

VALIDATE

Run structural checks: directory layout, manifest schema, component completeness, hook correctness, and cross-reference integrity.

/pqa:validate · /pqa:hooks-check
7

AUDIT

Deep quality audit: content depth, instruction clarity, security posture, conformance to the Claude Code plugin specification.

/pqa:audit
8

REPORT

Generate comprehensive QA reports and graded scorecards with per-category scores, trends, and actionable improvement targets.

/pqa:report · /pqa:scorecard
9

FIX

Auto-detect and repair common issues: missing files, broken cross-references, schema errors, incomplete MOAT structures.

/pqa:fix

Quality Gates

4 checkpoints that halt the pipeline until quality criteria are met

G1
After DESIGN
Agent definitions complete. Skill MOAT strategy approved. No orphan commands.
G2
After BUILD
Scaffold generated. All files exist. plugin.json valid. Hooks installed.
G3
After AUDIT
Quality score above threshold. No critical findings. Security scan clean.
G4
After FIX
All auto-fixable issues resolved. Final scorecard grade A- or above.

MOAT Methodology

Four pillars of plugin quality depth

M

Methods

Step-by-step procedures and workflows embedded in SKILL.md. Define how the skill operates, decision trees, and fallback behaviors.

O

Ontology

Domain knowledge references in references/ directories. Glossaries, standards, specs, and domain models the skill draws upon.

A

Assets

Concrete examples in examples/ directories. Input/output pairs, before/after samples, and edge case demonstrations.

T

Templates

Reusable templates and prompt patterns. Standardized output formats, scaffolding blueprints, and prompt engineering patterns.

Agent Responsibilities

Who does what in the PQA pipeline

Plugin Architect

IDEATE → PLAN → DESIGN

Owns concept generation, architecture planning, and component design. Creates the blueprint.

Spec Writer

SPECIFY

Authors formal specifications for commands, skills, hooks, and manifests. Bridges design to build.

Builder Agent

BUILD → FIX

Scaffolds directories, generates files, installs hooks, and applies auto-fixes post-audit.

QA Engineer

VALIDATE → AUDIT → REPORT

Validates structure, audits quality, generates scorecards, and enforces quality gates.
